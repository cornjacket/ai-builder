# Regression Test: Fibonacci

This test verifies that the AIDT+ pipeline correctly architects, implements, and
tests a Golang fibonacci package from a job specification.

---

## Directory Structure

```
fibonacci/
    gold/                           # committed — never visible to pipeline agents
        gold_fibonacci_test.go      # gold test (build tag: regression)
        go.mod                      # imports from work/fibonacci via replace
    work/                           # pipeline working directory (cwd for agents)
        JOB-fibonacci-demo.md.gold # committed — pristine task spec
        JOB-fibonacci-demo.md      # gitignored — restored by reset.sh
        fibonacci/                  # gitignored — generated by pipeline
        logs/                       # gitignored — generated by pipeline
        execution.log               # gitignored — generated by pipeline
        go.mod                      # committed — work module declaration
    reset.sh                        # resets work/ to initial state
    README.md                       # this file
```

### Why gold/ and work/ are separate

The pipeline's `--output-dir` is `work/`. The TESTER agent runs with `work/` as
its working directory and has no visibility into `gold/`. This prevents the
TESTER from reading or copying the gold test file instead of independently
deriving tests from the Acceptance Criteria. All regression tests must follow
this `gold/` / `work/` separation for the same reason.

---

## What This Tests

The pipeline is given `work/JOB-fibonacci-demo.md` as input. It must:

1. **ARCHITECT** — fill in the Design and Acceptance Criteria sections
2. **IMPLEMENTOR** — produce `work/fibonacci/fibonacci.go` implementing `Compute(n int) []int`
3. **TESTER** — verify the implementation against all acceptance criteria

After the pipeline completes, `gold/gold_fibonacci_test.go` is run to
independently verify the generated package against the gold test cases.

---

## Run Order

**Step 1 — Reset the test:**

```bash
tests/regression/fibonacci/reset.sh
```

Restores `work/JOB-fibonacci-demo.md` from the `.gold` file and removes all
generated artifacts from any previous run.

**Step 2 — Run the pipeline** (from repo root):

```bash
python3 ai-builder/orchestrator.py \
    --job tests/regression/fibonacci/work/JOB-fibonacci-demo.md \
    --output-dir tests/regression/fibonacci/work
```

**Step 3 — Run the gold test:**

```bash
cd tests/regression/fibonacci/gold && go test -tags regression ./...
```

The `//go:build regression` tag excludes the gold test from normal `go test`
runs. Pass `-tags regression` explicitly to run it.
