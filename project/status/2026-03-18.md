## Key Decisions

- **`no_history` is meaningless for internal agents** — internal roles never call
  `build_prompt()`, so the flag is omitted from the `LEAF_COMPLETE_HANDLER` entry.

- **Test Command belongs in the job doc, not the role prompt** — ARCHITECT knows the
  target language and repo conventions; TESTER should be a pure executor. Hardcoding
  `go test ./...` in `TESTER.md` was wrong.

- **TESTER floor is workload-dependent** — unlike LCH (which does almost nothing), TESTER
  actively reads the codebase. Its cached token floor scales with the size of the
  generated code, not a fixed constant.

- **Run 11 bundles two changes** — TESTER Test Command and LCH internal agent are tested
  together because the Test Command change requires ARCHITECT behaviour changes, making
  clean single-variable isolation impractical.
