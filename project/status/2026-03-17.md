## Key Decisions

- **Handlers use `"prompt": null` in machine JSON** — handler roles require runtime
  variables (TARGET_REPO, EPIC, etc.) in their prompts so they can't use static `.md`
  files. `null` signals the orchestrator to fall through to Python-generated prompts.
  Other roles load static files from `roles/`.

- **`--state-machine` auto-selection** — rather than requiring the flag for every run,
  the orchestrator infers the right machine from whether `--target-repo` is present.
  Existing invocations work without changes.

- **Run summary as a required output artifact** — `run-summary.md` (human-readable)
  and `run-metrics.json` (machine-readable) will be written to the output directory
  on `HANDLER_ALL_DONE`. Format mirrors the manually-produced platform-monolith summary:
  per-invocation table, per-role totals, token usage by role, decomposition tree,
  implementations table.
