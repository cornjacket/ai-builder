## Key Decisions

- **Networked monolith = one binary, one process** — multiple HTTP listeners
  start from a single `main.go`. The platform-monolith spec was wrong on first
  draft; corrected before second run.

- **ARCHITECT owns contract propagation** — in decompose mode, the Components
  table Description column is the only vehicle for passing the spec to
  design-mode ARCHITECT. It must carry the full API contract for HTTP-handling
  components, not a summary. Internal components get data models only.

- **Scripts over prompt logic** — all deterministic pipeline operations
  (complete, check stop-after, advance) are in scripts called by the TM agent.
  The orchestrator remains agnostic of the task management system.

- **Pipeline boundary = USER-TASK / USER-SUBTASK Task-type** — tree traversal
  stops when it reaches a human-owned node. The pipeline never marks USER-TASK
  or USER-SUBTASK nodes complete.

- **Regression test spec must be Oracle-reviewed before pipeline run** —
  captured in the SOP spec review checklist. A bad spec produces a wasted run.
