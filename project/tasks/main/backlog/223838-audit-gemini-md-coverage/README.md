## Goal

Ensure every `CLAUDE.md` in the committed codebase has a corresponding
`GEMINI.md` in the same directory, and that the content of each `GEMINI.md`
is current with its `CLAUDE.md` counterpart. Add a rule to `CLAUDE.md`
requiring this pairing to be maintained going forward.

## Context

Claude Code loads `CLAUDE.md` automatically; Gemini CLI loads `GEMINI.md`.
For any directory where behaviour guidance has been written, both files should
exist so that either agent can be used without missing instructions.

**Current state (2026-04-04):** The committed `CLAUDE.md`/`GEMINI.md` pairs
are in sync at the file level:

- `CLAUDE.md` ↔ `GEMINI.md` (repo root)
- `tests/regression/CLAUDE.md` ↔ `tests/regression/GEMINI.md`

Sandbox target `CLAUDE.md`/`GEMINI.md` files are generated by
`bootstrap/init-claude-md.sh` and `reset.sh` — they are gitignored and do
not need to be audited here.

**What this task should do:**

1. **Content audit** — compare each `GEMINI.md` against its `CLAUDE.md` pair.
   Identify any sections or rules present in `CLAUDE.md` that are absent or
   stale in `GEMINI.md`, and bring them in sync.

2. **Coverage rule** — add a rule to the root `CLAUDE.md`: whenever a
   `CLAUDE.md` is created or updated, the corresponding `GEMINI.md` in the
   same directory must be created or updated in the same commit.

3. **Verify `init-claude-md.sh`** — confirm that the script that generates
   target-repo `CLAUDE.md`/`GEMINI.md` files also stays in sync when new
   rules are added to the templates.
